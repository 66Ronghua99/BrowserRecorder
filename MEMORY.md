## Project Experience

### 2026-03-07 | Chrome Extension MVP

**经验沉淀**:

- Chrome Extension Manifest V3 需要使用 content script 注入
- 摄像头权限需要通过 `navigator.mediaDevices.getUserMedia()` 获取
- 悬浮窗需要使用 `position: fixed` 和高 z-index 确保覆盖网页内容
- 拖动逻辑需要计算鼠标偏移量防止跳动

### 2026-03-07 | 设置不生效问题

- **问题**: popup 控制面板的设置不生效（大小、显示/隐藏、圆角）
- **根因**: style.css 使用了 `!important`，但 JS 设置的是 inline style，被 CSS 覆盖
- **修复**: 使用 `style.setProperty(property, value, 'important')` 覆盖 CSS
- **预防**: 动态设置的样式需要用 setProperty + 'important'

### 2026-03-07 | 圆形和缩放优化

- **问题**: 圆形显示为椭圆，缩放以左上角为原点导致人像出画
- **修复**: 圆形模式下改为正方形尺寸；缩放改用 `transform-origin: center center`

### 2026-03-07 | 录屏慢放与音频拖长

- **问题**: 回放时视频像慢放，音频也有被拖长感，连续录制后更明显
- **根因**:
  - 文件在 Chrome 标签页内播放正常，问题主要出现在部分本地播放器对 WebM 的兼容性
  - WebM 在本地播放器上可能出现时长条异常或解码观感变慢
- **修复**:
  - 录制输出固定为 16:9 画布（1080p/900p/720p）保证跨平台分辨率一致
  - 导出格式改为 MP4 优先，浏览器不支持时回退 WebM
  - 保留回退日志，便于确认当前设备的编码支持情况
  - 麦克风初始化失败时不终止录制，自动降级为“仅系统音频/无音频”
- **预防**:
  - 回归测试需同时覆盖 Chrome 回放 + 本地播放器回放
  - 录制参数（尺寸、格式）显式可配置并持久化，避免隐式设备差异
  - 录制状态必须由录制链路本身写回存储（开始/停止），不要只依赖 popup 响应时机
  - 录制状态优先写 `storage.local`（实时 UI 状态），`storage.sync` 仅做兼容同步

## Global Constraints

- 插件必须在所有网站生效 (`<all_urls>`)
- MVP 不包含 AI、虚拟背景、录屏等功能
- 必须覆盖所有网页元素 (z-index: 2147483647)

### 2026-03-10 | 竖屏录制支持

- **问题**: 现有录制仅支持 16:9 横屏，无法满足小红书/抖音发布需求
- **方案**: 添加 3:4 (1080x1440) 竖屏尺寸 + 自动调整浏览器窗口
- **实现**:
  - RECORDING_SIZE_MAP 新增 `'1080p-3:4': { width: 1080, height: 1440 }`
  - 录制开始前检测竖屏尺寸，调用 `window.resizeTo()` 调整窗口
  - 录制结束后恢复原始窗口尺寸
  - 复用现有 MP4 优先导出逻辑
- **风险**: 浏览器可能拒绝调整窗口大小 → 回退提示用户手动调整
- **状态**: 已完成 ✅
